# Transformer And Low-IR

## 职责

Transformer 负责把 parser raw lowering 成统一 low-IR。

这一层只做两类事情：

- 把 dialect-specific raw 指令翻译成共享 low-IR 语义
- 建立排错与映射所需的 `LoweringMap`

这一层不负责：

- 重建 parser 协议事实
- 做 CFG、数据流、结构恢复
- 直接决定 AST 级语法糖

## 应消费的前层事实

Transformer 应消费：

- `ChunkHeader.dialect/version/layout`
- `RawProto` 树
- `RawInstr` typed view
- `RawConstPool` accessor
- `RawProto` / `RawChunk` extra accessor

Transformer 不应自己重建：

- opcode label
- operand decode
- extra-word 绑定关系
- child proto 槽位映射
- header bit layout

这些事实已经属于 Parser。

## 应优先复用的共享设施

### parser/raw accessor

如果 raw 层已经有 accessor，应优先通过 accessor 消费，而不是直接匹配 `Dialect*Extra`。

### 共享 lowering 状态

`src/transformer/dialect/lowering.rs` 提供：

- `PendingLoweringState`
- `PendingMethodHints`
- `WordCodeIndex`
- `resolve_pending_instr_with`

涉及 raw pc、target 回填、method hint、line hint 时，应先接到这里，而不是在各 dialect lowerer 各养一份状态机。

### PUC-Lua family helper

`src/transformer/dialect/puc_lua.rs` 负责 PUC-Lua 家族共性，例如：

- for-loop lowering 骨架
- call / return / tailcall 共享 emit
- env upvalue 相关 helper
- 跳转目标辅助计算

`lua52~55` 的版本文件应尽量只保留真正的方言差异。

### operand expect 宏

`src/transformer/operands.rs` 的 `define_operand_expecters!` 是 dialect lowerer 解析 operand 形状的正式共享设施。

如果只是“把 operand 解构成若干具名字段”，应优先复用它。

## 维护规范

### 1. 版本文件只保留语义差异

`lower.rs` 文件应优先表达：

- 这个 opcode 在 low-IR 里对应什么语义
- 这个方言与共享 family 的真正差异是什么

它不应继续承担：

- parser 协议解码
- lowering plumbing 样板
- opcode label / operand 大表

### 2. low-IR 不保留 dialect-specific instruction

某条 raw 指令进入 CFG 前，必须已经被 lowering 成共享 `LowInstr`。

### 3. LoweringMap 是正式契约

需要排错或回溯时，应通过 `LoweringMap`，而不是把 raw pc、line、origin 再塞进 low-IR 主体。

### 4. 语义事实优先前移

如果某个 dialect helper 经常在多个 lowerer 中平行出现，优先判断它是否属于：

- parser 协议事实
- 共享 lowering 状态
- PUC-Lua family 语义 helper

不要默认把它留在某个单独 `lower.rs` 里。

### 5. 错误边界应清晰

如果输入 raw 与该 dialect lowering 契约不符，应直接返回 `TransformError`，不要静默忽略或硬猜。

## 向后提供的事实

Transformer 向后正式提供：

- `LoweredChunk`
- `LoweredProto`
- `LowInstr`
- `LoweringMap`
- `InstrRef`
- `Reg` / `ProtoRef` / `ConstRef` / `UpvalueRef`
- `ValueOperand` / `ResultPack` / `ValuePack`

## 维护检查清单

修改 Transformer 时，应至少检查：

1. 这个事实是不是其实属于 Parser/raw。
2. 这个 helper 是否已经能放进 `dialect/lowering.rs` 或 `dialect/puc_lua.rs`。
3. 版本文件里是不是混进了大量 emit 样板或状态 plumbing。
4. 新增 low-IR 语义是否真的跨 dialect 共享。
5. `LoweringMap` 是否仍能稳定对齐 raw/low。
