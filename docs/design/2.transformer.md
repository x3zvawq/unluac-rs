# Transformer / Low-IR

> **职责**：把 dialect-specific raw 指令翻译成统一 `LowInstr`，建立 `LoweringMap`。
>
> **不负责**：CFG 切块、数据流分析、AST 语法选择。
>
> **例子**：`RawInstr::LoadK { dst: R0, k: 5 }` → `LowInstr::LoadConst { dst: Reg(0), src: ConstRef(5) }`

## 入口

```
src/transformer/mod.rs
  lower_chunk(state, context)      -- Transform 阶段入口，写回 LoweredChunk 槽位
  lower_raw_chunk(raw_chunk)       -- 显式事实 helper，按 dialect 版本分派到各 dialect lowerer
```

## 模块布局

```
src/transformer/
  mod.rs           入口 + 版本分派
  common.rs        统一 low-IR 类型（LowInstr / LoweredChunk / LoweredProto / LoweringMap 等）
  error.rs         TransformError
  operands.rs      define_operand_expecters! 宏
  debug.rs         dump_lir / format_low_instr
  dialect/
    lowering.rs         共享 lowering 状态：PendingLoweringState / WordCodeIndex / resolve_pending_instr_with
    puc_lua.rs          PUC-Lua 家族语义 helper：for-loop / call / return / tailcall / env-upvalue / jump target
    lua51/ ... lua55/   各版本 lower.rs（只保留真正的方言差异）
    luajit/             lower.rs
    luau/               lower.rs
```

## 数据流

```
RawChunk
  └─ lower_raw_chunk
       └─ dialect::lua54::lower_chunk(raw_chunk)
            ├─ puc_lua.rs helpers
            ├─ lowering.rs PendingLoweringState
            └─ LoweredChunk {
                 main: LoweredProto {
                   instrs: Vec<LowInstr>,
                   lowering_map: LoweringMap,
                   children: Vec<LoweredProto>,
                 }
               }
```

## 关键类型

| 类型 | 说明 |
|---|---|
| `LowInstr` | 统一低级指令，不含 dialect-specific 信息 |
| `LoweredChunk` / `LoweredProto` | 翻译后的模块树 |
| `LoweringMap` | raw pc → low-IR InstrRef 映射，排错回溯的正式接口 |
| `InstrRef` / `Reg` / `ConstRef` / `UpvalueRef` / `ProtoRef` | 稳定索引类型 |
| `ValueOperand` / `ResultPack` / `ValuePack` | 多值 carrier 表达 |

## 应优先复用的共享设施

| 设施 | 用途 |
|---|---|
| `lowering.rs::PendingLoweringState` | raw pc / target 回填 / method hint / line hint，多 dialect 共用 |
| `lowering.rs::raw_pc_at` / `next_raw_pc` | word-len-bearing dialect 的 raw pc 查询，避免各版本 lowerer 复制索引样板 |
| `puc_lua.rs` | RK / `k` bit operand 转换、immediate 条件操作数、PUC-Lua jump target 计算、for-loop lowering 骨架、call / return / tailcall emit、env upvalue helper、`ADDI+MMBINI` family 恢复规则 |
| `define_operand_expecters!` | operand 解构成具名字段，不要手写 match arm |
| parser raw accessor | 已有 accessor 就优先消费，不直接匹配 `Dialect*Extra` |

## 维护规范

1. **版本文件只保留语义差异**：`lower.rs` 应只表达"这个 opcode 在 low-IR 里对应什么"和"与共享 family 的真正差异"。
2. **low-IR 不保留 dialect-specific instruction**：进入 CFG 前必须已经翻译成统一 `LowInstr`。
3. **`LoweringMap` 是正式契约**：排错 / 回溯通过 `LoweringMap`，不把 raw pc / line / origin 再塞进 low-IR 主体。
4. **语义事实优先前移**：如果某个 helper 在多个 lowerer 里平行出现，判断它是否属于 parser 协议事实 / 共享 lowering 状态 / PUC-Lua family 语义 helper。
5. **错误边界清晰**：输入 raw 与 dialect 契约不符时直接返回 `TransformError`，不静默忽略。
6. **同名 opcode 也要按版本语义处理**：例如 `LOADNIL` 在 Lua 5.1 使用结束寄存器语义，而 Lua 5.2/5.3/5.4/5.5 的 `B` 是从 `A` 开始的偏移量。lowering 时应先确认 raw 指令协议，不要因为 opcode 名相同就复用相同 range 计算。
7. **method hint 按寄存器写入失效**：`SELF/NAMECALL` 与最终 `CALL` 之间可能夹着参数求值调用，例如 `obj:method(f())`；pending method hint 只能在 callee/self slot 或返回值写入范围被覆盖时失效，不能因为中间出现普通 `CALL` 就全局清空。

## 排错指引

| 症状 | 检查点 |
|---|---|
| low-IR 指令语义与源码不符 | `transformer/dialect/puc_lua.rs` family 规则是否正确匹配 opcode 组合 |
| method hint 丢失 | `lowering.rs::PendingMethodHints` 是否正确传递 |
| for-loop 变量错位 | `puc_lua.rs` for-loop lowering 骨架 |
| 想看 low-IR | `unluac --dialect lua54 --debug --dump transformer --stop-after transformer <file>` |
