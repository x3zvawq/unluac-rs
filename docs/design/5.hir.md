# HIR

> **职责**：消费前层所有事实，做正式恢复决策，产出 AST 可以直接消费的语义树。
>
> **不负责**：重建前层事实、AST sugar、最终命名。
>
> **例子**：`StructureFacts::BranchCandidate { header, merge, arms }` → `HirIf { cond, then_block, else_block }`

## 入口

```
src/hir/mod.rs
  analyze_hir(state, options, timings) -> HirModule
```

## 结构化 loop 的局部出口

HIR 的 loop lowering 只消费 `StructureFacts` 给出的 loop/branch 候选，但在具体降树时会处理两类局部边界：

- break exit pad 可以包含 cleanup 语句、短路条件或小型 if 结构；lowering 复用已有 branch plan，把 pad 降成 `if ... end; break`，不会在 loop 层重新识别短路。
- while 体内的分支如果同时存在 break 出口和本轮共享收尾块（例如 `elseif ... then ... break end; i = i + 1`），分支会先停在共享收尾块，再由外层 loop body 统一消费一次，避免两条分支臂重复 visit 同一个 continuation。
- `while true` 型 jump latch 的前缀语句属于循环体尾部；HIR 需要把这段 tail 追加到 body，而不是为 Lua 5.1 目标制造 continue/goto。

输入形状：`while cond do if a then ... elseif b then ... break end; step = step + 1 end`  
输出形状：`while cond do if a then ... elseif b then ... break end; step = step + 1 end`

### 新增 pass 步骤

1. 在 `simplify/PASS_DESCRIPTORS` 数组添加 `PassDescriptor { name, phase, depends_on, invalidates }`。
2. 在 `simplify_hir` 的 `match index` 分支添加调用。
3. 在 `simplify/` 新建对应文件，优先使用 `walk.rs::HirRewritePass` 或 `visit.rs::HirVisitor`。

## 应优先复用的共享设施

| 设施 | 用途 |
|---|---|
| `analyze/structure/overrides.rs` | entry/phi override 管理，不在各路径平行复制 |
| `analyze/structure/rewrites.rs` | 共享表达式重写，不在各路径平行复制 |
| `analyze/short_circuit/` | 短路决策 / guard lowering / recovery，DAG 事实来自 StructureFacts |
| `simplify/walk.rs::HirRewritePass` | 后序 rewrite，所有子节点先完整递归再执行当前 hook |
| `simplify/visit.rs::HirVisitor` | 只读 collector，收集 label/temp/binding 一类事实 |
| `simplify/mention.rs` | local/temp 提及与 temp 写入查询；pass 不各自复制 visitor |
| `simplify/local_shapes.rs` | local 声明/左值基础形状判断；pass 不把通用 helper 挂在某个语义 pass 上 |
| `simplify/temp_touch.rs` | temp 引用集合、逐语句 temp refs 缓存、控制头部消费、非局部控制流查询，以及进入子作用域时的前后缀保护集 |
| `expr_safety.rs` | 表达式是否会观察求值序的共享谓词；analyze / simplify 不各自维护副作用口径 |

`simplify/carried_locals/` 内部继续按关注点拆分：`mod.rs` 只做后序遍历与 block 调度，`adjacent.rs` 只处理结构化相邻 `seed local + empty carried local`，`boundary.rs` 只收敛显式 `label/goto` 边界快照的 carried binding 等价类，`handoffs.rs` 只判断并执行普通 seed/update handoff，`seeds.rs` 只解析和重写 seed 语句，`reads.rs` 只收集 carried binding 读取事实，`binding.rs` 只提供 local/temp carried binding 表示和 rewrite pass，`prune.rs` 只裁掉已证明由本次合并制造出的冗余 self-assign。

`simplify/decision/` 内部按 Decision DAG 责任拆分：`mod.rs` 保留 pass 入口、通用 reduce、value/condition collapse 与 graph remap，`specialize.rs` 只做沿路径已知 test 结果传播和节点 interning，`helpers.rs` 只处理普通 Lua 逻辑表达式局部形状，`synthesize/` 负责把可证明等价的纯 Decision 合成为可读逻辑表达式，`eliminate.rs` 只负责最终残余 Decision 的 block/stmt 遍历，`eliminate_materialize.rs` 只负责表达式抽取和值物化，`eliminate_state.rs` 只保存物化期间追加 synthetic local 的状态。

`simplify/locals/` 内部保留单一 `locals` pass owner：`locals.rs` 决定哪些 temp 可以提升、何时复用 closure 已捕获的 local；`branch_merge.rs` 只汇总 if/else 两侧都 fallthrough 的 temp 写回候选；`rewrite.rs` 只消费已确认的 `TempId -> LocalId` 映射改写表达式、左值和 closure capture；`param_alias.rs` 只处理函数入口参数机械别名的后置收敛。

**walker 契约**：所有子节点必须先完整递归；不能因为前一子树 `changed=true` 就短路跳过后续子树；当前节点 hook 也必须总是执行。

## 维护规范

1. **HIR 决策依赖 StructureFacts**：branch 合流 / loop binding / short-circuit leaf 等证据从 StructureFacts 读，不从 dataflow 裸存储重新推断。
2. **fallback 只处理"仍然没有 owner"的情况**：不重新覆盖 branch / loop / short-circuit 已经接管的事实。
3. **simplify 只做 HIR 内部收敛**：不替 StructureFacts 漏掉的事实兜底。
4. **branch-value collapse 在 HIR**：`if cond then x=v; goto L end; x=d; label L`、`local x; if cond then x=a else x=b end` 与 nil-only fallback alias 这类值壳，应在 HIR 先收回结构化 if/else，再用 `Decision->Expr` 或保守 nil-check 改写处理；Readability 只处理语法外观，不重新判断这类值合流语义。
5. **branch stop 分层**：`branch_stop_for_region` 描述整个 branch 回到外层 region 的边界；单条 arm 降低时可使用更窄的 arm stop，避免一条臂提前消费外层 stop 之后的共享尾块。典型形状：`if guard then ... elseif handled then return ... end; return ...`。
6. **carried-locals 保守**：结构化相邻 `seed local + empty carried local`、fallback label/goto 握手都由 HIR 认回同一状态身份；只有 seed 在后续不再可观察、carried 不被闭包捕获、且写回形状可证明时才改写。`assign(t0,t1) = call(...)` 多值赋值不能按 zip 截断。
7. **locals pass 边界**：只物化真正"局部、一次性"的 temp；loop/branch-carried update（`t1 = t1 * t0`）不应被提升成新 local。
   入口参数别名是 locals 的后置收敛步骤：只处理函数体入口 `local L = param` / `local L; L = param`、且参数原值不会在 `L` 写入后继续被读取的形状；它不替 phi/loop state 重新分类，也不把循环累加器写回参数。普通 `local; assign` 的语法合并留给 Readability `statement_merge`。
8. **dialect-neutral canonicalization**：Lua 5.4/5.5 的 `>> -1` / `<< -2` 应在 HIR 收回 `<< 1` / `>> 2`。
9. **短路的 single-eval 只覆盖未物化节点**：`if A and B then` 这类 branch-exit 短路在入口 header 会先通过 `lower_block_prefix` 物化前缀语句；若为了后续被吞掉的 header 需要 single-eval 展开，只能展开那些不会单独 lower 的节点，入口 header 仍应引用已物化 temp，避免 `local x = f(); if f() and ...` 这类重复求值。value-merge 短路同理：`assert(a() == 1 and b() == 2)` 这类布尔合流应在 HIR 恢复成 `LogicalAnd/LogicalOr`，但 single-eval 不能把已经物化的外部调用或作为取表 base 的调用结果重新展开成 `build()[1]()`。
10. **表达式物化要保持求值序**：Lua 会先求 call callee，再求参数；bytecode 因此常见 `t_f = assert; t_arg = <短路表达式>; t_f(t_arg)`。HIR 可以把它恢复成 `assert(<短路表达式>)`，但不能只把 callee 内联成 `t_arg = <短路表达式>; assert(t_arg)`，否则全局读取会被挪到参数副作用之后。反过来，如果源码形状是 `local v = source(); sink(v)` 或 `local v = source; sink(v)`，也不能在 callee 已物化到 call 后继续把更早的调用、全局/表读取、运算或短路参数折回 `sink(...)`，否则这些可被元方法或副作用观察到的求值会被挪到 callee 读取之后。同理，loop header prefix 中先执行的 `side()` 不能被折到条件里位于更早求值的 `guard()` 之后；遇到这类重排时应保留 prefix 并回退成 `while true + if break`。
11. **试探式 lowering 必须可回滚**：conditional reassign、value merge、loop continue/break、cross escape 等 branch 快捷路径，以及 numeric/generic for preheader 这种会先试降 loop body 的路径，如果失败都必须恢复 visited / override / active loop / 输出语句状态，再交给下一种结构化策略。
12. **shared continuation 不是普通 fallthrough**：`if cond then gated end; shared` 这类结构只有在 gated 的所有非终止路径都到 shared，或在当前 active loop 边界上 break/continue，才可恢复成共享 continuation；loop escape 必须显式留在 gated block 内。
    当 repeat header 被临时当作普通 branch 降低，且其 region 内存在无 label、非 branch/loop header 的多臂共享线性 tail，但部分路径需要先跳到 region merge 来跳过 tail 时，HIR 应合成 `repeat ... until true` fence：跳向 merge 的路径降成 fence 内的 `break`，tail 仍作为单一 continuation 降在 fence 末尾，不复制 tail，也不退回 label/goto。普通 branch-value merge 不套这个 fence，避免把结果 phi 隔在 fence 内外两侧。
13. **loop exit pad 可以是小型结构**：numeric/generic/while 的 break exit pad 不只可能是线性 cleanup，也可能是 `if cleanup_cond then cleanup end` 后再到 post-loop。loop context 需要记录 pad 消费的所有 block，避免 coverage 阶段误以为 pad body 未被结构化。
14. **非空 continue target 不能简单继续**：generic/numeric for 的 tail header 往往是非空 continue target。短路 branch 若 `truthy` 正好跳到这个 target，应先取反并交换出口，恢复成 `if cond then body end`；若 `falsy` 跳到 target，则可以按 if-then guard 处理。repeat/while 仍保持保守，避免把循环条件边界吞进短路表达式。
15. **被短路吞掉的 header prefix 要先重写**：扩展 branch-exit short-circuit 后，后续被消费 header 的 prefix temp 不会再单独 lower。HIR 需要用已有 prefix temp override 把这些 temp 内联进合成条件；仍残留不可解析 prefix temp 时才回退，避免生成引用未定义 temp 的条件。
16. **loop state 初值允许使用 preheader 出口表达式**：generic/numeric loop 可能紧跟在一串已结构化 branch-value merge 后面，preheader 边上的同一寄存器会保留多个原始 def。只要这是进入 loop 前的 block 出口值，就可以用 `expr_for_reg_at_block_exit(preheader, reg)` 作为 state 初值，而不是在 loop state 里重新拆这些 branch merge。
17. **closure capture 的 sticky local 以词法槽位为边界**：`locals` pass 可以把 closure 已捕获的同槽位后续 def 写回原 local，但这个“同槽位”必须包含 `close from rX` 形成的 epoch。close 后 VM 复用同一个寄存器号时已经是新的词法槽位，不能继续写回已关闭 upvalue；同一条赋值 RHS 捕获当前槽位时，也要先激活 sticky 身份再决定该赋值是否复用原 local。若 closure 捕获的是非参数、非入口 local、且捕获点没有 reaching value 的父级槽位，bindings 需要先分配空 local，并把同 epoch 后续 def 直接写回该 local，避免分支写入因没有普通读者而被清理。
18. **fallback label/goto 的 loop-carried 更新可以认回同一 binding**：当无法安全结构化成 `while/repeat` 时，`carried-locals` 仍可在 HIR 内保守合并 `state = next; ::entry:: ... next = state + 1; if cond(next) then goto handoff end` 这类回边交棒。前提是 `next` 没有前缀读取、handoff 紧贴一个可被外部路径直接进入的 label、后续回边确实跳回 handoff label，且 `next` 的定义只依赖这一组 carried binding；这样可以恢复单一源码 local，同时不重新触发“goto 跳进 local 作用域”问题。
19. **while header prefix 不能默认丢弃**：`while-like` header 里位于 branch 前的指令只有在确实属于条件计算时才可以被内联进条件并省略；如果 prefix 定义不参与条件、但在 header 出口仍 live（例如 guard 前声明一个会被循环体 closure 捕获的 `local x`），HIR 必须回退成 `while true do <prefix>; if break-cond then break end; <body> end`。这样保留每轮迭代 guard 前初始化的词法槽位，避免 closure 结果临时和被捕获 local 混成同一绑定。
20. **closure capture 需要稳定父级绑定**：`temp-inline` 不能删除跨出 `if`、`block`、`unstructured` 等嵌套结构后仍被 closure capture 引用的 temp；这类 temp 不只是普通表达式临时值，还会成为子 proto 的 upvalue provenance，删除后会让 naming 指向不存在的父级 synthetic local。
21. **exit-only loop state 可以从 preheader 取初值**：`found = false; for ... do found = true; break end; if found then ... end` 这类形状的 exit phi，所有 incoming 可能都来自 loop body 或 break pad，CFG 上没有单独的“循环外 predecessor”。HIR 应在确认 incoming 全部属于 loop/break-pad 后，用 preheader 出口值作为 state 初值，让 break pad 写回同一状态槽位。
22. **for-binding 作用域消费 StructureFacts**：generic/numeric for 的自然 loop blocks 不包含 `break` 前的 cleanup tail，但 Lua 词法作用域仍覆盖这些 tail。Structure 的 `LoopCandidate.binding_scope_blocks` 已包含被 header 支配且尚未到达正常 loop exit 的提前退出块，bindings 层只据此注册 for 变量可见性，避免 `table.remove(list, i); break` 里的 `i` 退回成未初始化 temp/local。
23. **短路出口的透明 jump pad 属于短路结构**：StructureFacts 的短路候选只记录条件 header 和语义 truthy/falsy 出口；Lua 5.1 字节码可能在某个出口前插入只含 `JMP` 的空 pad。HIR 构建 branch plan 时应把这类无 phi、无前缀语句、唯一流向短路出口的 pad 标记为已消费，但不能把它当作条件 header 参与 prefix 重写。
24. **多入口 loop state 先看共同原始初值**：没有唯一 preheader 的 while-like loop 可能来自 `if a or b then while ... end end`，多个外部 predecessor 都跳到同一个 header。HIR 计算 header/exit state 初值时应先用 raw incoming defs 判定是否是同一个表达式；若这些入口还共享同一个可写 lvalue，则 loop state target 应复用这个入口槽位，而不是新建只在分支内部赋值的 phi temp。这样跳过 loop 的外层路径仍能看到源码初值。
25. **带 closure 的 terminal tail 不能复制**：共享 terminal return block 若前缀里创建 closure，应作为单一 continuation 消费，不能为了避免重访 terminal block 把整段 tail 克隆进多条分支。closure capture 依赖父级词法槽位身份；复制 raw `CLOSURE` 会让同一个 child proto 在 HIR 中出现多个互不兼容的 capture 来源。
26. **generic phi 不猜测值**：short-circuit value merge 恢复失败时应显式保留 unresolved；没有 branch/loop/short-circuit owner 的 generic phi 只消费 Structure 给出的 `GenericPhiSource`。`IdomExit` 才可用对应 block 出口表达式物化，`Unresolved` 必须继续暴露 unresolved，不能在 HIR 重新猜测某个支配块近似值。
27. **路径覆盖不把未知回环当成功**：branch shared continuation、loop continuation 和 stop/escape 截断这类 all-paths 判定，只接受闭包显式分类的边界；遇到 reducible loop header 时只检查该 loop 的显式 exits，递归中遇到尚未被 loop/branch owner 分类的回环时应视为未证明，不能把它当作所有路径都已到达目标边界。
28. **value-merge header 不能随便取第一个候选**：branch-lowering 的 value-merge 快捷路径只允许消费同一 header 下唯一候选，或同一棵短路树上的多 result phi；若同一 header 下出现多棵不同 value-merge 树，应退回普通 branch/merge lowering，避免某个 phi 被候选选择器静默丢掉。普通 BranchValueMerge 的 HIR 快捷消费同样只能在同 header 唯一候选时启用；`conditional-reassign` 只能消费单个 result phi；同树多输出必须退回普通 branch，一次性保留所有赋值，不能把同一条件拆成多段补写。
