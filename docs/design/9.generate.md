# Generate

## 目标

Generate 负责把：

- readability 之后的稳定 AST
- `NameMap`

转换成最终 Lua 源码文本。

这一层是纯输出层，不再做结构恢复、语法糖决策、可读性折叠或命名分配。

## 当前规则

- Generate 不修改 AST 结构
- Generate 不重新分配名字
- Generate 消费 `ReadabilityResult`、`NameMap`、target dialect 和生成选项
- Generate 负责最终的文本布局、缩进、换行和转义
- Generate 需要按“当前函数上下文”查询 `NameMap`，而不是把所有 binding id 当作模块级全局唯一键
- Generate 不为前层已经丢失的源码痕迹兜底；如果 AST 里没有 `global *` 这类节点，Generate 不能自行发明

## 输入

```text
GenerateInput
  ast: ReadabilityResult
  names: NameMap
  target_dialect
  options: GenerateOptions
```

这里的 `NameMap` 应当和 AST 的稳定函数身份对齐，
这样 Generate 在递归进入嵌套函数时，才能切换到正确的名字上下文。

这里强调 `ReadabilityResult`，是为了固定阶段边界：

- AST build 负责把 HIR 语义落成合法语法树
- Readability 负责把语法树收敛成更接近源码的稳定形态
- Generate 只消费最终稳定 AST，不再重新折叠、抽 sugar 或回头改结构

## Generate Options

```text
GenerateOptions
  indent_width
  max_line_length
  quote_style
  table_style
  conservative_output

QuoteStyle
  PreferDouble
  PreferSingle
  MinEscape

TableStyle
  Compact
  Balanced
  Expanded
```

默认值：

- `indent_width = 4`
- `max_line_length = 100`
- `quote_style = MinEscape`
- `table_style = Balanced`
- `conservative_output = true`

规则：

- 空白和布局策略由用户决定，Generate 提供稳定默认值
- 长函数参数列表、表构造器布局、`return` 多值断行都受 `max_line_length` 和 `table_style` 控制
- `conservative_output = true` 时优先输出更稳妥、更少语法糖惊喜的文本

## Doc 模型

Generate 采用轻量文档模型，而不是直接 visitor 拼字符串。

```text
Doc
  Text
  Line
  SoftLine
  Concat
  Indent
  Group
```

规则：

- AST 先 lowering 成 `Doc`
- 最终再由 renderer 根据 `GenerateOptions` 渲染成字符串
- 这样可以把语义输出和布局决策分开
- 可以在 `generate` 模块内部通过 visitor / trait lowering 遍历 AST，但节点本身不直接持有“输出字符串”的方法

推荐的内部结构可以是：

```text
generate
  doc
  emit
  render
```

其中：

- `emit` 负责 AST -> Doc
- `render` 负责 Doc -> String
- AST 节点本身仍保持纯语法数据，不混入布局逻辑

## 输出职责

Generate 负责：

- 关键字和标识符输出
- 代码块缩进
- 语句换行和空行策略
- 表构造器文本布局
- 字符串字面量转义
- 数字字面量文本格式
- `goto` / label 输出
- 函数声明和函数表达式输出
- `global` / `global<const>` / `global *` 输出

## 运算符优先级与括号

Generate 必须显式维护 Lua 运算符优先级，而不是依赖 AST 当前长相“刚好不出错”。

至少需要覆盖：

- 一元运算：`not / - / ~ / #`
- 幂：`^`
- 乘除模整除：`* / % //`
- 加减：`+ -`
- 拼接：`..`
- 位移：`<< >>`
- 位与：`&`
- 位异或：`~`
- 位或：`|`
- 比较：`< <= == ~= > >=`
- 逻辑与或：`and / or`

规则：

- 只有在当前子表达式文本放入父表达式后不会改变 Lua 求值顺序时，才能省略括号
- `and / or / .. / ^` 这类结合方向敏感的位置，必须按 Lua 实际结合律决定是否保留括号
- 保守多打一层括号是允许的；改变语义绝对不允许
- `conservative_output = true` 时应当偏保守，而不是追求“最少括号”

## 布局策略

- block 结构默认多行输出
- 小型表构造器在 `Compact/Balanced` 下可单行
- 大型表构造器在 `Balanced/Expanded` 下优先多行
- 长参数列表和长返回值列表根据 `max_line_length` 自动换行
- 布局决策必须稳定，不依赖哈希顺序或遍历偶然性

## 字符串与字节

Generate 必须尊重前面层保留下来的字符串与名字信息：

- 对可安全显示的文本直接输出
- 对不安全字符使用合法 Lua 转义
- 不依赖 parser 阶段的原始字节直接回填源码

## global 语法边界

Generate 只负责把 AST 中已经明确存在的 global 语法节点输出出来。

当前需要支持：

- `global x = ...`
- `global<const> x = ...`
- `global a, b = ...`
- `global *`
- `global<const> *`

这里的规则是：

- `global *` / `global<const> *` 必须在 AST 中有显式节点或显式 wildcard binding 表达后，Generate 才能输出
- 如果前层因为缺少运行时证据而没有恢复出 zero-runtime-trace 的 global 声明，Generate 不能擅自补猜
- `global<const> *` 只允许在 target dialect 支持 `global_const` 时出现

## Consumes / Provides

- Consumes: `ReadabilityResult`, `NameMap`, `GenerateOptions`, target dialect
- Provides: Lua 源码文本
