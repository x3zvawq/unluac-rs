# Generate

## 目标

Generate 负责把：

- target-dialect-aware AST
- `NameMap`

转换成最终 Lua 源码文本。

这一层是纯输出层，不再做结构恢复、语法糖决策、可读性折叠或命名分配。

## 当前规则

- Generate 不修改 AST 结构
- Generate 不重新分配名字
- Generate 消费 AST、`NameMap`、target dialect 和生成选项
- Generate 负责最终的文本布局、缩进、换行和转义
- Generate 需要按“当前函数上下文”查询 `NameMap`，而不是把所有 binding id 当作模块级全局唯一键

## 输入

```text
GenerateInput
  ast
  names: NameMap
  target_dialect
  options: GenerateOptions
```

这里的 `NameMap` 应当和 AST 的稳定函数身份对齐，
这样 Generate 在递归进入嵌套函数时，才能切换到正确的名字上下文。

## Generate Options

```text
GenerateOptions
  indent_width
  max_line_length
  quote_style
  table_style
  conservative_output

QuoteStyle
  PreferDouble
  PreferSingle
  MinEscape

TableStyle
  Compact
  Balanced
  Expanded
```

默认值：

- `indent_width = 4`
- `max_line_length = 100`
- `quote_style = MinEscape`
- `table_style = Balanced`
- `conservative_output = true`

规则：

- 空白和布局策略由用户决定，Generate 提供稳定默认值
- 长函数参数列表、表构造器布局、`return` 多值断行都受 `max_line_length` 和 `table_style` 控制
- `conservative_output = true` 时优先输出更稳妥、更少语法糖惊喜的文本

## Doc 模型

Generate 采用轻量文档模型，而不是直接 visitor 拼字符串。

```text
Doc
  Text
  Line
  SoftLine
  Concat
  Indent
  Group
```

规则：

- AST 先 lowering 成 `Doc`
- 最终再由 renderer 根据 `GenerateOptions` 渲染成字符串
- 这样可以把语义输出和布局决策分开

## 输出职责

Generate 负责：

- 关键字和标识符输出
- 代码块缩进
- 语句换行和空行策略
- 表构造器文本布局
- 字符串字面量转义
- 数字字面量文本格式
- `goto` / label 输出
- 函数声明和函数表达式输出

## 布局策略

- block 结构默认多行输出
- 小型表构造器在 `Compact/Balanced` 下可单行
- 大型表构造器在 `Balanced/Expanded` 下优先多行
- 长参数列表和长返回值列表根据 `max_line_length` 自动换行
- 布局决策必须稳定，不依赖哈希顺序或遍历偶然性

## 字符串与字节

Generate 必须尊重前面层保留下来的字符串与名字信息：

- 对可安全显示的文本直接输出
- 对不安全字符使用合法 Lua 转义
- 不依赖 parser 阶段的原始字节直接回填源码

## Consumes / Provides

- Consumes: target-dialect-aware AST, `NameMap`, `GenerateOptions`, target dialect
- Provides: Lua 源码文本
