# Generate

## 职责

Generate 负责把 Readability 之后的稳定 AST 与 `NameMap` 渲染成最终 Lua 源码文本。

这一层是纯输出层。

## 入口与模块

### 对外入口

- `src/generate.rs`
  - `generate_chunk`

### 共享模型

- `src/generate/common.rs`
- `src/generate/error.rs`

### Doc 与渲染

- `src/generate/doc.rs`
- `src/generate/render.rs`

### 发射阶段

- `src/generate/emit/mod.rs`
- `src/generate/emit/expr.rs`
- `src/generate/emit/stmt.rs`
- `src/generate/emit/syntax.rs`
- `src/generate/emit/names.rs`

## 应消费的前层事实

Generate 应消费：

- Readability 之后的 AST
- `NameMap`
- `GenerateOptions`
- `AstTargetDialect`

Generate 不应再决定：

- 结构恢复
- 命名策略
- 可读性 sugar
- 缺失的语法节点

## 应优先复用的共享设施

### AST -> Doc -> String

`Generate` 的正式分层是：

1. `emit/*`：AST -> Doc
2. `render.rs`：Doc -> String

新的输出规则应优先落在这两层正确位置，不要在一个函数里同时做语义选择与文本拼接。

### names / syntax helper

- `emit/names.rs` 负责读取 `NameMap`
- `emit/syntax.rs` 负责标点、关键字、通用语法发射

涉及名字选择或共享语法片段时，应优先补这里，而不是在 `expr.rs` / `stmt.rs` 各写一份。

## 维护规范

### 1. Generate 不补前层丢失信息

如果 AST 中没有：

- `global *`
- `global<const>`
- `SingleValue`
- `local <close>`

Generate 不应自行发明。

### 2. precedence 由 Generate 统一维护

运算符优先级与括号规则应集中在表达式发射路径，不应让调用方依赖 AST “刚好长成安全形状”。

### 3. 布局与语义分开

同一条规则里如果同时涉及：

- 语义选择
- 行宽折行
- 缩进

应优先把布局部分留给 `Doc` / `render`，不要在语义发射里直接拼硬编码字符串。

### 4. NameMap 始终按函数上下文读取

Generate 进入嵌套函数时，应切换到正确的函数上下文，而不是把 binding id 当作全模块唯一。

## 向后提供的事实

Generate 向外提供：

- `GeneratedChunk`
- 最终源码文本

## 维护检查清单

修改 Generate 时，应至少检查：

1. 这是语义决定，还是纯输出规则。
2. 该逻辑是不是应该先在 AST / Readability / Naming 表达。
3. 该共享输出规则是不是应落到 `emit/names.rs` 或 `emit/syntax.rs`。
4. 布局策略是否继续通过 `Doc` / `render` 实现，而不是直接拼字符串。
