# CFG / GraphFacts / Dataflow

> 这三层共享同一组底层模块（`src/cfg/`），但职责严格分开：
> - **CFG**：把线性 low-IR 组织成 basic block + 控制流边
> - **GraphFacts**：从 CFG 计算稳定图事实（支配 / 回边 / 自然循环）
> - **Dataflow**：从 CFG + GraphFacts 计算 def/use / phi / liveness / side-effect

## 入口

```
src/cfg/mod.rs
  build_cfg_proto(lowered_proto)           -> CfgGraph
  analyze_graph_facts(cfg_graph)           -> GraphFacts
  analyze_dataflow(proto, cfg, facts, children) -> DataflowFacts
```

## 模块布局

```
src/cfg/
  mod.rs          入口 + 所有类型导出
  build.rs        CFG 切块（LowInstr 控制流终结语义 → BasicBlock + Edge）
  graph.rs        支配树 / 后支配树 / 回边 / 自然循环分析
  dataflow.rs     analyze_dataflow 骨架
  debug.rs        dump_cfg / dump_graph_facts / dump_dataflow
  common/
    cfg.rs        Cfg / CfgGraph / BasicBlock / BlockRef / CfgEdge / EdgeKind
    graph.rs      DominatorTree / PostDominatorTree / NaturalLoop / GraphFacts
    dataflow.rs   DataflowFacts / DefId / PhiId / InstrEffect / SideEffectSummary 等
    storage.rs    紧凑容器（CompactSet / ValueSetRef 等）
  dataflow/
    effects.rs    副作用分析
    fixed.rs      fixed-point 框架
    liveness.rs   活跃性分析
    phi.rs        phi candidate 识别
    values.rs     def / use / reaching values
```

## 数据流

```
LoweredProto
  └─ build_cfg_proto
       └─ CfgGraph { cfg: Cfg, children: Vec<CfgGraph> }
            └─ analyze_graph_facts(cfg_graph)
                 └─ GraphFacts { dominator, post_dominator, backedges, natural_loops }
                      └─ analyze_dataflow(proto, cfg, graph_facts, children)
                           └─ DataflowFacts { defs, phi_candidates, use_values,
                                             liveness, side_effects, open_packs }
```

---

## CFG 层

### 关键类型

| 类型 | 说明 |
|---|---|
| `Cfg` | block + edge 集合，提供所有 block/edge 查询 |
| `CfgGraph` | Cfg + 子 proto CfgGraph 树 |
| `BasicBlock` | `[InstrRange, BlockKind]`，不含指令内容 |
| `BlockRef` / `EdgeRef` | 稳定索引 |
| `EdgeKind` | `Fallthrough` / `Branch(true/false)` / `Jump` / `Return` 等 |

### 关键查询（`Cfg` 方法）

- `branch_edges(block)` — 条件跳转两条边
- `reachable_successors(block)` — 可达后继
- `unique_reachable_successor(block)` — 唯一后继（仅 fallthrough / jump）
- `reachable_predecessors(block)` — 可达前驱
- `can_reach(from, to)` — 可达性查询
- `can_reach_within(from, to, boundary)` — 带边界的可达性

### 维护规范

- 切块规则只放在 `build.rs`，edge kind 语义只放在 `common/cfg.rs`。
- 后层需要稳定 block/edge 查询时，优先回到 `Cfg` 补 query，不要在 Structure 或 HIR 重写一份。

---

## GraphFacts 层

### 关键类型

| 类型 | 说明 |
|---|---|
| `GraphFacts` | dominator / post-dominator / backedges / natural loops 等 |
| `DominatorTree` / `PostDominatorTree` | 支配树 / 后支配树 |
| `NaturalLoop` | header + body blocks |

### 维护规范

- 同类图分析只算一次，再派生到多个事实。
- Structure / HIR 不应自己再做一份 dominator / frontier / region 分析。
- 后层频繁需要的图结论应补成 `GraphFacts` query 或共享事实。

---

## Dataflow 层

### 关键类型

| 类型 | 说明 |
|---|---|
| `DataflowFacts` | 所有数据流事实的统一容器 |
| `DefId` | 一条赋值/phi 的稳定 id |
| `PhiId` / `PhiCandidate` / `PhiIncoming` | phi 候选及来源 |
| `InstrEffect` / `SideEffectSummary` | 指令副作用 |
| `InstrReachingDefs` / `InstrReachingValues` | 到达定义 / 到达值 |
| `InstrUseDefs` / `InstrUseValues` | use-def / use-value 链 |

### 关键查询（`DataflowFacts` 方法）

- `def_reg(def_id)` — 该定义写哪个寄存器
- `def_block(def_id)` — 该定义在哪个 block
- `def_instr(def_id)` — 该定义在哪条指令
- `instr_def_for_reg(block, instr, reg)` — 某条指令某寄存器的定义
- `latest_local_def_in_block(block, reg)` — block 内最后一条可见定义
- `phi_use_count(phi_id)` — phi 被使用次数
- `phi_used_only_in_block(phi_id, block)` — phi 是否只在某个 block 内消费

### 维护规范

- HIR / Structure 不应直接把 `dataflow.defs` / `use_values` / `phi_candidates` 当裸存储扫描。
- 后层频繁需要的"语义化查询"应优先补到 `DataflowFacts`。
- 热路径优先稠密结构、轻量集合、共享索引，不在后层重复 clone。

---

## 三层共同维护规则

- Structure 和 HIR 只能通过公开 query/事实消费这三层结果。
- 某个查询被多个消费者重复实现时，优先回收到 `src/cfg/common/`。
- `common/storage.rs` 负责紧凑容器，主路径性能设施优先放这里。

## 排错指引

| 症状 | 检查点 |
|---|---|
| block 数量 / 边数量不对 | `build.rs` 切块逻辑，边 kind 是否正确 |
| dominator 关系错误 | `graph.rs` 支配树算法 |
| phi candidate 识别遗漏 | `dataflow/phi.rs` join point 判定 |
| reaching def 不准确 | `dataflow/fixed.rs` fixed-point 框架 |
| 想看 CFG | `unluac --debug --target-stage cfg <file>` |
| 想看 GraphFacts | `unluac --debug --target-stage graph-facts <file>` |
| 想看 Dataflow | `unluac --debug --target-stage dataflow <file>` |
