# AST

> **职责**：把 HIR 落成合法 Lua 语法树，完成 AST 内部可读性整形，并把稳定 binding 身份映射成最终名字。
>
> **不负责**：补 Structure/HIR 缺失事实、改变已恢复语义、文本布局。
>
> **例子**：`HirIf { cond, then_block, else_block }` → `AstIf { cond: AstExpr, then_block: AstBlock, else_block: Option<AstBlock> }`

## 入口

```
src/ast/mod.rs
  lower_ast(hir_module, target_dialect, generate_mode) -> Result<AstModule, AstLowerError>
  analyze_ast_stage(state, context) -> Result<(), DecompileError>
```

`analyze_ast_stage` 是 decompile pipeline 的 AST 阶段入口：内部依次执行 AST build、
readability 和 naming。Strict 模式按请求方言 lowering，Permissive 模式会先放宽
lowering 能力，让 Generate 阶段统一产出目标方言不支持特性的 warning。

可读性整形和命名是 AST 内部子步骤；`--dump ast` 会分段输出 AST / Readability / Naming：

```
src/ast/readability/mod.rs
  make_readable(state, context) -> ()
  make_readable_module(module, target, options, timings, dump_passes) -> AstModule
src/ast/naming/mod.rs
  collect_naming_evidence(hir_module) -> Result<NamingEvidence, NamingError>
  assign_names_with_evidence(ast, hir, evidence, options) -> Result<NameMap, NamingError>
```

```

  build/mod.rs    lower_ast 骨架

    analysis.rs           HIR → AST lowering 分析
    exprs.rs              表达式 lowering
    patterns/
      syntax.rs           合法语法化规则
      shape.rs            语义保真形状规则
  readability/mod.rs  make_readable 骨架 + pass 描述符 + 调度
  readability/    所有可读性 pass（见 7.readability.md）

  traverse.rs     共享遍历
  debug.rs        dump_ast / dump_ast_snapshot
```

## 数据流

```
HirModule + AstTargetDialect + GenerateMode
  └─ lower_ast（显式事实 helper）/ lower_ast_for_generate（stage 入口）
       └─ build/analysis.rs + build/exprs.rs + build/patterns/
            └─ AstModule { protos: Vec<AstProto>, ... }
                  └─ make_readable / make_readable_module
                       └─ AstModule（经过可读性整形）
                          └─ naming::assign_names
                             └─ NameMap
```

## 关键类型（`src/ast/common.rs`）

| 类型 | 说明 |
|---|---|
| `AstModule` / `AstProto` | 模块树 |
| `AstBlock` / `AstStmt` / `AstExpr` | 基本结构 |
| `AstIf` / `AstWhile` / `AstRepeat` / `AstNumericFor` / `AstGenericFor` | 控制流 |
| `AstLocalDecl` / `AstAssign` / `AstCallStmt` / `AstReturn` / `AstGoto` | 语句 |
| `AstTargetDialect` / `AstDialectCaps` | 目标方言 + 能力控制 |
| `DecompileDialect` | 主反编译 dialect 枚举，AST 层在 `common.rs` 为它补充关键字与能力集语义 |
| `AstLocalAttr` / `AstGlobalAttr` | `<const>` / `<close>` 属性 |
| `AstLocalBinding` / `AstGlobalBinding` | binding 身份 |
| `AstSyntheticLocalId` | Readability 引入的合成 local |
| `AstFunctionDecl` / `AstLocalFunctionDecl` / `AstFunctionExpr` | 函数节点 |
| `ReadabilityOptions` | AST readability 与 HIR temp-inline 共享的源码形状阈值 |
| `NameMap` / `FunctionNameMap` / `NameInfo` | AST naming 产出的最终名字映射 |

## 应优先复用的共享设施

| 设施 | 用途 |
|---|---|
| `AstDialectCaps` | 语法合法性校验，集中控制，不散落到各节点处理 |
| `build/patterns/syntax.rs` | 合法语法化规则 |
| `build/patterns/shape.rs` | 语义保真形状规则（纯可读性 sugar 不属于这里） |
| `readability/walk.rs` / `visit.rs` / `traverse.rs` | AST readability pass 共享遍历 |
| `naming/ast_facts.rs` / `lexical.rs` | AST 词法边界和 binding 可见性 |

## 维护规范

1. **build 只负责合法语法化与语义保真**：
   - 适合：`goto+close` 合法词法块、`SingleValue` 语义保真节点、保守 local/global 节点、hoisted temp 声明。
   - hoisted temp 按 block 内首次出现顺序排布，避免 `statement_merge` 因顺序错位无法下沉。
   - 不适合：method / function sugar、installer IIFE、field access sugar、local merge / stmt merge（这些属于 Readability）。
2. **target dialect 能力通过 `AstDialectCaps` 集中控制**：不在各节点处理代码散落版本判断。
3. **readability 只做 AST 形状收敛**：method sugar、field access sugar、local/statement merge 等放在 AST readability；branch value 语义合流仍回 HIR。
4. **naming 只产出名字映射**：不改变 AST 结构，不发明前层缺失 binding。
5. **binding 身份保持稳定**：AST build / readability 不把 `ParamId` / `LocalId` / `TempId` 等提前改成字符串。
6. **Generate 不替 AST 兜底**：`global *` / `global<const>` / `local <close>` / `SingleValue` 等必须在 AST 显式建模。

## 向后提供的事实

合法 Lua 语法树 / 目标 dialect capability 下允许的节点 / 稳定 binding 身份 / AST readability 收敛后的形状 / `NameMap`。

## 排错指引

| 症状 | 检查点 |
|---|---|
| 语法节点类型错误（如 while vs repeat） | `build/analysis.rs` HIR 节点分派 |
| `<close>` / `<const>` 属性丢失 | `build/patterns/syntax.rs` |
| hoisted temp 声明顺序错误 | `build/mod.rs` temp hoist 逻辑 |
| dialect 约束未生效 | `AstDialectCaps` 校验逻辑 |
| method/field/local merge 不理想 | `readability/` 对应 pass |
| 名字冲突或 debug hint 丢失 | `naming/strategy.rs` / `naming/allocate.rs` / `naming/evidence/` |
| 想看 AST / readability / naming | `unluac --debug --dump ast --stop-after ast <file>` |
