# AST

> **职责**：把 HIR 落成合法 Lua 语法树，保持目标方言约束。
>
> **不负责**：可读性 sugar、最终命名、文本布局。
>
> **例子**：`HirIf { cond, then_block, else_block }` → `AstIf { cond: AstExpr, then_block: AstBlock, else_block: Option<AstBlock> }`

## 入口

```
src/ast/mod.rs
  lower_ast(hir_module, target_dialect, generate_mode) -> Result<AstModule, AstLowerError>
```

可读性整形入口（被 pipeline 通过 `resolve_output_plan` 调用）：

```
src/ast/readability.rs（作为 ast 子模块）
  make_readable(module, target, options, timings, dump_passes) -> AstModule
```

## 模块布局

```
src/ast/
  mod.rs          入口 + 所有类型导出
  common.rs       所有 AST 节点类型
  error.rs        AstLowerError
  features.rs     collect_ast_features（feature 收集）
  build.rs        lower_ast 骨架
  build/
    analysis.rs           HIR → AST lowering 分析
    exprs.rs              表达式 lowering
    patterns/
      syntax.rs           合法语法化规则
      shape.rs            语义保真形状规则
  readability.rs  make_readable 骨架 + pass 描述符 + 调度
  readability/    所有可读性 pass（见 7.readability.md）
  pretty.rs       共享 pretty 设施（被 generate 引用）
  traverse.rs     共享遍历
  debug.rs        dump_ast / dump_readability / dump_ast_snapshot
```

## 数据流

```
HirModule + AstTargetDialect
  └─ lower_ast
       └─ build/analysis.rs + build/exprs.rs + build/patterns/
            └─ AstModule { protos: Vec<AstProto>, ... }
                  └─ make_readable(module, target, options, timings, dump_passes)
                       └─ AstModule（经过可读性整形）
```

## 关键类型（`src/ast/common.rs`）

| 类型 | 说明 |
|---|---|
| `AstModule` / `AstProto` | 模块树 |
| `AstBlock` / `AstStmt` / `AstExpr` | 基本结构 |
| `AstIf` / `AstWhile` / `AstRepeat` / `AstNumericFor` / `AstGenericFor` | 控制流 |
| `AstLocalDecl` / `AstAssign` / `AstCallStmt` / `AstReturn` / `AstGoto` | 语句 |
| `AstTargetDialect` / `AstDialectCaps` | 目标方言 + 能力控制 |
| `AstDialectVersion` | Lua51~55 / LuaJit / Luau |
| `AstLocalAttr` / `AstGlobalAttr` | `<const>` / `<close>` 属性 |
| `AstLocalBinding` / `AstGlobalBinding` | binding 身份 |
| `AstSyntheticLocalId` | Readability 引入的合成 local |
| `AstFunctionDecl` / `AstLocalFunctionDecl` / `AstFunctionExpr` | 函数节点 |

## 应优先复用的共享设施

| 设施 | 用途 |
|---|---|
| `AstDialectCaps` | 语法合法性校验，集中控制，不散落到各节点处理 |
| `build/patterns/syntax.rs` | 合法语法化规则 |
| `build/patterns/shape.rs` | 语义保真形状规则（纯可读性 sugar 不属于这里） |

## 维护规范

1. **build 只负责合法语法化与语义保真**：
   - 适合：`goto+close` 合法词法块、`SingleValue` 语义保真节点、保守 local/global 节点、hoisted temp 声明。
   - hoisted temp 按 block 内首次出现顺序排布，避免 `statement_merge` 因顺序错位无法下沉。
   - 不适合：method / function sugar、installer IIFE、field access sugar、local merge / stmt merge（这些属于 Readability）。
2. **target dialect 能力通过 `AstDialectCaps` 集中控制**：不在各节点处理代码散落版本判断。
3. **binding 身份保持稳定**：AST 阶段不把 `ParamId` / `LocalId` / `TempId` 等提前改成字符串。
4. **Generate 不替 AST 兜底**：`global *` / `global<const>` / `local <close>` / `SingleValue` 等必须在 AST 显式建模。

## 向后提供的事实

合法 Lua 语法树 / 目标 dialect capability 下允许的节点 / 稳定 binding 身份 / 语义保真的保守节点与属性。

## 排错指引

| 症状 | 检查点 |
|---|---|
| 语法节点类型错误（如 while vs repeat） | `build/analysis.rs` HIR 节点分派 |
| `<close>` / `<const>` 属性丢失 | `build/patterns/syntax.rs` |
| hoisted temp 声明顺序错误 | `build.rs` temp hoist 逻辑 |
| dialect 约束未生效 | `AstDialectCaps` 校验逻辑 |
| 想看 AST | `unluac --debug --target-stage ast <file>` |
