# 总览

## 目标

本项目的目标是构建一套面向多 Lua dialect 的反编译基础设施。

当前明确覆盖的 family：

- PUC-Lua 5.x
- LuaJIT
- 后续可扩展到其他 Lua 方言

当前阶段的重点不是直接实现完整反编译，而是把各层职责、输入输出和依赖关系先定稳。这样开始编码时，每一层知道自己该依赖什么、该产出什么、哪些事不该抢做。

## 总体流水线

1. Parser
2. Transformer
3. CFG
4. GraphFacts
5. Dataflow
6. StructureFacts
7. HIR
8. AST
9. Readability
10. Naming
11. Generate

## 设计原则

- 不提前抹平差异。不同 dialect、不同版本的差异先完整保留在前端。
- 不提前丢信息。原始字节流里能稳定提取出的信息都应保留。
- 统一发生在语义层，不发生在 raw 层。
- 先保真，再优化。正确和可回退优先于好看。
- 分层单向依赖。后层消费前层结果，不反向侵入前层职责。

## 当前分层边界

- Parser：把字节流解析成保真的 raw 结构。
- Transformer：把 raw 结构 lowering 成统一的 low-IR。
- CFG：把线性 low-IR 切成 basic block 并建立控制流图。
- GraphFacts：从 CFG 计算支配关系、回边、自然循环等图事实。
- Dataflow：在 CFG 和 GraphFacts 基础上计算 SSA-like、def-use、活跃信息和副作用摘要。
- StructureFacts：从图事实和数据流事实中提取结构候选与跳转保留需求。

## 当前实现约定

- 各 dialect 独立 parser，实现层通过 feature 控制。
- Transformer 不保留 dialect-specific low-instr 残留。
- CFG 保留完整图，但后续主分析默认只跑 reachable 子图。
- Dataflow 只做单 proto 内分析，不做过程间 call 副作用传播。
- StructureFacts 只产候选，不直接产最终源码结构。

## 性能约束

目标是让整条流水线在大输入上仍保持稳定性能。至少应考虑处理 `10^5` 级别指令序列时，构图和分析不能出现明显的多重重复遍历瓶颈。

当前性能原则：

- 尽量使用基于索引的新类型和 `Vec`/arena 存储，而不是在主路径大量依赖 `HashMap`
- 对 block、instr、def、edge、proto 等主对象使用稳定整数索引，确保 O(1) 查询
- `instr_to_block`、`preds`、`succs`、`low_to_raw`、`raw_to_low` 这类映射应前置构建，避免后续重复扫描
- 图分析和数据流分析优先使用 worklist + RPO，而不是反复全图暴力遍历
- `RegSet`、活跃集合、可达集合优先考虑 bitset/稠密集合结构
- open pack 走独立通道，避免伪装成大量固定寄存器导致集合膨胀
- 语义层和生成层要尽量复用前面已经算出的事实，不重复回到底层扫描

需要缓存的典型信息：

- `InstrRef -> BlockRef`
- `BlockRef -> preds/succs`
- `InstrRef -> InstrEffect`
- `InstrRef -> DefId[]`
- `DefId -> use sites`
- `LoweringMap`
- Naming 阶段的 binding -> final string

## 依赖关系

- Parser provides：`RawChunk`, `RawProto`, `RawInstr`
- Transformer consumes：Parser
- Transformer provides：`LoweredProto`, `LoweringMap`, `LowInstr`
- CFG consumes：Transformer
- CFG provides：`Cfg`
- GraphFacts consumes：CFG
- GraphFacts provides：支配树、frontier、回边、自然循环
- Dataflow consumes：Transformer, CFG, GraphFacts
- Dataflow provides：`InstrEffect`, `SideEffectSummary`, `DefId`, `reaching_defs`, `phi_candidates`
- StructureFacts consumes：CFG, GraphFacts, Dataflow
- StructureFacts provides：branch/loop/short-circuit/goto/scope 候选
- HIR consumes：CFG, GraphFacts, Dataflow, StructureFacts
- HIR provides：结构化语义树、变量身份、fallback 结果
- AST consumes：HIR, target dialect context
- AST provides：target-dialect-aware 语法树
- Readability consumes：AST, target dialect context, 副作用和绑定约束
- Readability provides：更接近源码的稳定 AST
- Naming consumes：稳定 AST、绑定引用、作用域信息
- Naming provides：最终字符串命名
