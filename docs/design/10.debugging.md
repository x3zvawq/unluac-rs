# 调试实现 Code Map

> 本文只回答“调试能力在哪里实现、扩展时该改哪”。命令怎么用见 `docs/debug.md`。

## 总体流向

```text
CLI 参数
   -> DecompileOptions / DebugFilters
   -> stages 调度表决定要不要 dump / timing / pass-dump
   -> 各层 debug.rs 负责把本层状态渲染成文本
   -> stderr 输出调试信息，stdout 保持正常结果
```

## 核心文件

| 位置 | 职责 |
| --- | --- |
| `src/decompile/options.rs` | 调试相关选项进入 `DecompileOptions` 的入口 |
| `src/decompile/pipeline.rs` | 创建一次调用的状态与 stage 上下文 |
| `src/decompile/stages.rs` | 外层阶段表同时绑定 run / dump 函数，按阶段执行 dump / timing，并把 pass dump 请求转交给 HIR / AST readability |
| `src/debug.rs` | `DebugFilters`、`ProtoDepth` 等跨层共享类型，以及 `define_stage_dump!` stage dump 宏 |
| `src/debug/focus.rs` | `--proto` / `--proto-depth` 的共享聚焦算法 |
| `src/debug/colorize.rs` | ANSI 着色与终端输出辅助 |
| `src/timing.rs` | `--timing` 所需的阶段打点与汇总 |
| `src/hir/simplify/mod.rs` | `--dump-pass` 的 before/after 快照调度 |

## 各层 dump 入口

| 层 | 文件 | 主要函数 |
| --- | --- | --- |
| parser | `src/parser/debug.rs` | `dump_parser` |
| transformer | `src/transformer/debug.rs` | `dump_lir` |
| structure | `src/structure/debug.rs` + `src/structure/cfg/debug.rs` | `dump_structure`（内含 CFG / GraphFacts / Dataflow / StructureFacts 分段） |
| hir | `src/hir/debug.rs` | `dump_hir` / `dump_proto_snapshot` |
| ast | `src/ast/debug.rs` + `src/ast/naming/debug.rs` | `dump_ast`（内含 AST / Readability / Naming 分段） / `dump_ast_snapshot` |
| generate | `src/generate/debug.rs` | `dump_generate` |

## 共享语义

- `DebugFilters { proto, proto_depth }` 是所有分层 dump 的统一过滤接口。
- `compute_focus_plan` 把 proto 树映射成 “可见 / elided / 跳过” 三态，避免每层各写一套聚焦逻辑。
- AST / Readability 的 proto id 与 parser / HIR 对齐，保证同一个 `--proto N` 可以跨层比对。
- generate 层不做 proto 裁剪，因为裁剪后的 Lua 文本会变成非法语法。

## `--dump-pass` 代码路径

1. CLI 把 pass 名和过滤条件写入 `DecompileOptions`。
2. `src/decompile/stages.rs` 把 pass 名列表传给 HIR / AST readability 入口。
3. `src/hir/analyze/mod.rs` 组装 HIR `PassDumpConfig`；`src/ast/readability/mod.rs` 直接消费 AST pass 名列表。
4. HIR / AST pass 在前后抓取快照，并只在实际变更时输出。
5. HIR 快照由 `src/hir/debug.rs` 渲染；AST readability 快照由 `src/ast/debug.rs` 渲染。

## 扩展调试设施时看这里

1. 新增 stage dump：只给外层 `DecompileStage` 增加 dump。内部子步骤需要观察时，放进所属外层 dump 的分段里，不新增公开 stage 名。
2. 新增带 proto 概念的层：不要自写过滤规则，直接接入 `src/debug/focus.rs`。
3. 新增 pass-dump 能力：优先复用现有快照格式，不要另起一套输出协议。
4. 新增调试参数：入口放 `src/decompile/options.rs`，执行编排放 `src/decompile/stages.rs` 或对应业务层入口。
5. 若只是调某层业务逻辑，不要把业务事实塞进 debug 层；debug 层只负责观测，不负责修复。
