# Structure

> **职责**：从 low-IR 构建控制流、图事实、数据流事实，并把这些事实翻译成源码恢复候选。
>
> **不负责**：最终 HIR 语法决策、AST 语法糖、最终命名。
>
> **例子**：`branch_edges(B3) + phi_candidates_in(B5)` → `BranchCandidate { header: B3, merge: B5, arms: [B4a, B4b] }`

## 入口

```
src/structure/mod.rs
  analyze_structure_stage(state, context)      -- 依次写回 CfgGraph / GraphFacts / DataflowFacts / StructureFacts
```

CFG、GraphFacts、Dataflow 和 StructureFacts 是 Structure 内部固定子步骤；pipeline 外层只维护 `structure` 一个 stage / dump。

## 模块布局

```
src/structure/
  mod.rs          Structure 层入口 + 类型导出
  cfg/            CFG / GraphFacts / Dataflow 构建与调试
    build.rs        CFG 切块
    graph.rs        支配 / 后支配 / 回边 / 自然循环
    dataflow/mod.rs 数据流分析骨架
    common/         CfgGraph / DataflowFacts / GraphFacts 等共享类型
    dataflow/       def-use / phi / liveness / side-effect
    debug.rs        CFG / GraphFacts / Dataflow dump 片段
  common.rs       结构候选类型
  analyze.rs      StructureFacts 构建骨架
  helpers.rs      region / entry / exit / predecessor / reducible 共享查询
  phi_facts.rs    phi → 结构候选翻译
  branches.rs     branch 候选识别
  branch_values.rs  branch value merge 候选识别
  loops.rs        loop 候选识别
  goto.rs         GotoRequirement 识别
  regions.rs      region block 收集
  scope.rs        ScopeCandidate 识别
  short_circuit/  短路条件与值合流候选
  debug.rs        dump_structure
```

## 数据流

```
LoweredProto
  └─ cfg::build_cfg_graph
       └─ CfgGraph
          └─ cfg::analyze_graph_facts
             └─ GraphFacts
                └─ cfg::compute_dataflow_facts
                   └─ DataflowFacts
                      └─ analyze_structure_proto
                         └─ StructureFacts
```

## 关键类型

| 类型 | 说明 |
|---|---|
| `CfgGraph` / `Cfg` / `BasicBlock` / `EdgeKind` | block、边和基础控制流查询 |
| `GraphFacts` / `DominatorTree` / `NaturalLoop` | 支配、后支配、回边和自然循环 |
| `DataflowFacts` / `DefId` / `PhiId` / `InstrEffect` | def-use、phi、活跃性和副作用 |
| `StructureFacts` | 所有源码恢复候选的统一容器 |
| `BranchCandidate` / `LoopCandidate` / `ShortCircuitCandidate` | HIR 消费的结构候选 |
| `GotoRequirement` / `ScopeCandidate` / `GenericPhiMaterialization` | 必须保留的控制流约束和 fallback phi 物化候选 |

## 维护规范

1. **图和数据流事实属于 Structure**：后层需要稳定 block/edge/dataflow 查询时，优先补到 `src/structure/cfg/common/`，不要在 HIR 或 AST 重扫容器。
2. **结构证据应尽量前移**：HIR 需要的 phi 分类、merge arm、loop binding、short-circuit leaf 等事实优先补进 `StructureFacts`。
3. **候选与最终决策分开**：Structure 只回答“像不像 branch/loop/short-circuit”，最终语法选择交给 HIR。
4. **共享实现优先**：region、phi、短路和图查询先复用已有 helper；同类形状漏收时扩展 owner，不平行新建 pass。
5. **irreducible 判定看入口目标**：同一个 loop header 可以有多条外部前驱边；只有外部入口落到 region 内多个不同 block 时，才标记为 irreducible 并要求 goto。
6. **debug 分段不等于架构层**：`structure` dump 会按 CFG / GraphFacts / Dataflow / StructureFacts 分段输出，但外层仍只有 Structure。

## 排错指引

| 症状 | 检查点 |
|---|---|
| block / edge 数量不对 | `cfg/build.rs` 切块逻辑和 edge kind |
| dominator / natural loop 错误 | `cfg/graph.rs` |
| reaching def / phi candidate 错误 | `cfg/dataflow/` |
| branch 合流 block 错误 | `helpers.rs` region exit 逻辑 |
| loop 类型判断错误 | `loops.rs` |
| 短路候选丢失 | `short_circuit/branch_exit.rs` |
| phi 归属错误 | `phi_facts.rs` |
| 想看中间事实 | `unluac --debug --dump structure --stop-after structure <file>` |
