# Naming

## 职责

Naming 负责把 AST 中稳定存在的绑定身份映射成最终字符串名字。

它输出 `NameMap`，但不改变 AST 结构。

## 入口与模块

### 对外入口

- `src/naming.rs`
  - `collect_naming_evidence`
  - `assign_names`
  - `assign_names_with_evidence`

### 共享模型

- `src/naming/common.rs`
- `src/naming/error.rs`

### 证据层

- `src/naming/evidence/raw.rs`
- `src/naming/evidence/capture.rs`

### 命名实现

- `src/naming/ast_facts.rs`
- `src/naming/lexical.rs`
- `src/naming/hints.rs`
- `src/naming/strategy.rs`
- `src/naming/allocate.rs`
- `src/naming/assign.rs`
- `src/naming/support.rs`
- `src/naming/validate.rs`

## 应消费的前层事实

Naming 应消费：

- Readability 之后的稳定 AST
- `NamingEvidence`
- 目标命名模式与选项

Naming 不应消费：

- HIR 结构恢复细节
- CFG / Dataflow / StructureFacts
- 原生 `TempId`

如果 Readability 之后仍有原生 temp，属于前层边界未收敛。

## 应优先复用的共享设施

### evidence builder

新增命名证据时，应优先放在：

- `evidence/raw.rs`
- `evidence/capture.rs`

而不是直接在 `assign.rs` 或 `strategy.rs` 塞特殊分支。

### AST facts 与 lexical scope

涉及词法作用域、binding 可见性、函数上下文时，应优先复用：

- `ast_facts.rs`
- `lexical.rs`

### 策略与分配分离

- `strategy.rs` 负责候选名来源
- `allocate.rs` 负责冲突消解与槽位分配
- `assign.rs` 负责把这些结果组织成 `NameMap`

新的命名规则应尽量落在正确分层里，不要在 `assign.rs` 混合所有逻辑。

## 维护规范

### 1. Naming 面向最终 binding 集合

Naming 只处理最终保留下来的：

- `ParamId`
- `LocalId`
- `UpvalueId`
- synthetic local

### 2. 证据是辅助，不是正确性前提

`NamingEvidence` 可以改善结果，但不能成为 Naming 成功的必要条件。

没有 debug info 时，Naming 仍应稳定工作。

其中 `capture provenance` 不只覆盖“子闭包直接捕获父 local/upvalue”。
如果 HIR closure capture 指向的是父函数里的 temp-backed 绑定，evidence 也应继续保留
这条来源链，并在 `strategy.rs` 里解析回父函数已经稳定存在的 synthetic local 名字；
不要把这类“capture 来自临时槽位”的情况静默降级成新的裸 upvalue 名。

### 3. 作用域从 AST 自己推导

Naming 的词法边界来自 AST，而不是依赖额外的独立 scope dump。

### 4. 函数上下文是正式边界

`NameMap` 按函数上下文组织，不把所有 binding id 当作模块级单一平铺表。

Generate 在递归进入函数时，也应切换到对应函数的 NameMap。

## 向后提供的事实

Naming 向 Generate 正式提供：

- `NameMap`
- `FunctionNameMap`
- `NameInfo`

## 维护检查清单

修改 Naming 时，应至少检查：

1. 这个命名线索是不是应该先通过 `NamingEvidence` 输入。
2. 这是不是词法作用域问题，应回 `lexical.rs`。
3. 这是不是候选名来源问题，应回 `strategy.rs`。
4. 这是不是冲突分配问题，应回 `allocate.rs`。
5. 这个改动是否在偷偷改变 AST 结构；如果是，应回 Readability 或 AST。
