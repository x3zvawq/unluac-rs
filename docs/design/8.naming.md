# Naming

> **职责**：把 AST 中稳定存在的 binding 身份映射成最终字符串名字，输出 `NameMap`，不改变 AST 结构。
>
> **不负责**：改变 AST 结构、做结构恢复。
>
> **例子**：`LocalId(3)` + debug hint `"index"` → `NameMap[func][LocalId(3)] = "index"`

## 入口

```
src/naming.rs
  collect_naming_evidence(hir_module) -> Result<NamingEvidence, NamingError>
  assign_names_with_evidence(ast, hir, evidence, options) -> Result<NameMap, NamingError>
  assign_names(ast, hir, options)   -- 无 evidence 快捷入口
```

## 模块布局

```
src/naming/
  common.rs       NameMap / FunctionNameMap / NameInfo / NamingEvidence / NamingOptions / NamingMode
  error.rs        NamingError
  evidence/
    raw.rs        从 HirProto debug hint 提取证据（param / upvalue / local / temp）
    capture.rs    从 closure capture 推导 upvalue 来源
  ast_facts.rs    词法作用域 / binding 可见性分析（消费 AST）
  lexical.rs      词法作用域构建与查询
  hints.rs        debug hint 解析
  strategy.rs     候选名来源策略
  allocate.rs     冲突消解与槽位分配
  assign.rs       组装最终 NameMap
  support.rs      共享辅助
  validate.rs     命名合法性校验
  debug.rs        dump_naming
```

## 数据流

```
HirModule ──→ collect_naming_evidence ──→ NamingEvidence
                                               │
Readability 后的 AstModule ─────────────────→ assign_names_with_evidence
                                               │
                                           NameMap { func_id → FunctionNameMap { binding_id → NameInfo } }
```

## 应优先复用的共享设施

| 设施 | 用途 |
|---|---|
| `evidence/raw.rs` | 从 `HirProto.param_debug_hints` / `upvalue_debug_hints` / `local_debug_hints` / `temp_debug_locals` 提取，不直接访问 `RawProto` |
| `evidence/capture.rs` | closure capture → upvalue 来源链（含 temp-backed 绑定的情形） |
| `ast_facts.rs` / `lexical.rs` | 词法作用域 / binding 可见性，所有词法边界从 AST 推导 |
| `strategy.rs` | 候选名来源（改变候选策略在这里） |
| `allocate.rs` | 冲突消解（改变分配规则在这里） |

## 维护规范

1. **Naming 只处理最终保留的 binding**：`ParamId` / `LocalId` / `UpvalueId` / synthetic local；`TempId` 不应出现（属于前层边界未收敛）。
2. **证据是辅助，不是正确性前提**：无 debug info 时仍应稳定工作。
3. **capture provenance 覆盖 temp-backed 绑定**：如果 HIR closure capture 指向 temp-backed 绑定，evidence 应保留来源链并在 strategy 里解析回 synthetic local 名字；不要静默降级成裸 upvalue 名。
4. **词法边界来自 AST**：不依赖独立 scope dump。
5. **`NameMap` 按函数上下文组织**：Generate 递归进入嵌套函数时切换到对应 NameMap，不作模块级平铺。

## 向后提供的事实

`NameMap` / `FunctionNameMap` / `NameInfo`。

## 排错指引

| 症状 | 检查点 |
|---|---|
| 变量名全是 `v1` / `v2` | `evidence/raw.rs` debug hint 提取是否命中 |
| 闭包 upvalue 名丢失 | `evidence/capture.rs` capture provenance |
| 名字冲突 | `allocate.rs` 冲突消解逻辑 |
| 名字不合法（Lua 关键字等） | `validate.rs` |
| 想看 Naming 结果 | `unluac --debug --target-stage naming <file>` |
